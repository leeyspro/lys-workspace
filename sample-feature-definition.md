## 1. 기능 개요

### 1.1 기능명

비밀번호 재설정

### 1.2 기능 목적

사용자가 로그인 비밀번호를 잊어버린 경우, 가입된 이메일 인증을 통해 본인 여부를 확인하고 새로운 비밀번호를 설정할 수 있도록 한다.

### 1.3 적용 범위

- PC Web
- Mobile Web
- App WebView

### 1.4 관련 메뉴

- 로그인 화면
- 비밀번호 찾기 화면
- 이메일 인증 안내 화면
- 새 비밀번호 설정 화면
- 비밀번호 재설정 완료 화면

## 2. 사용자 시나리오

### 2.1 기본 시나리오

1. 사용자는 로그인 화면에서 `비밀번호 찾기`를 선택한다.
2. 사용자는 가입 이메일을 입력한다.
3. 시스템은 입력된 이메일의 가입 여부를 확인한다.
4. 가입된 이메일이면 비밀번호 재설정 링크를 발송한다.
5. 사용자는 이메일의 링크를 선택한다.
6. 사용자는 새 비밀번호를 입력하고 저장한다.
7. 시스템은 비밀번호를 변경하고 완료 화면을 노출한다.
8. 사용자는 새 비밀번호로 로그인할 수 있다.

### 2.2 예외 시나리오

- 가입되지 않은 이메일을 입력한 경우, 보안을 위해 가입 여부를 직접 노출하지 않는다.
- 인증 링크가 만료된 경우, 재요청 안내 화면을 노출한다.
- 이미 사용된 인증 링크로 접근한 경우, 만료와 동일하게 처리한다.
- 새 비밀번호가 정책에 맞지 않는 경우, 입력 필드 하단에 오류 메시지를 노출한다.

## 3. 화면별 기능 정의

### 3.1 로그인 화면

| 항목 | 정의 |
| --- | --- |
| 진입 조건 | 사용자가 로그인 화면에 접근 |
| 기능 위치 | 비밀번호 입력 영역 하단 |
| 버튼명 | 비밀번호 찾기 |
| 동작 | 선택 시 비밀번호 찾기 화면으로 이동 |

### 3.2 비밀번호 찾기 화면

| 항목 | 정의 |
| --- | --- |
| 화면 목적 | 사용자의 가입 이메일을 입력받아 재설정 링크 발송 |
| 입력값 | 이메일 |
| 필수 여부 | 필수 |
| 기본 상태 | 이메일 입력 필드, `인증 메일 보내기` 버튼 비활성화 |
| 활성화 조건 | 이메일 형식이 유효한 경우 버튼 활성화 |
| 성공 동작 | 인증 메일 발송 후 이메일 인증 안내 화면으로 이동 |
| 실패 동작 | 서버 오류 발생 시 토스트 메시지 노출 |

#### 이메일 입력 정책

| 조건 | 메시지 |
| --- | --- |
| 미입력 | 이메일을 입력해 주세요. |
| 이메일 형식 오류 | 올바른 이메일 형식으로 입력해 주세요. |
| 100자 초과 | 이메일은 100자 이하로 입력해 주세요. |

#### 보안 정책

가입되지 않은 이메일이어도 사용자에게는 동일하게 `인증 메일을 발송했어요.` 화면을 노출한다. 단, 실제 메일은 발송하지 않는다.

## 4. 비밀번호 정책

### 4.1 새 비밀번호 입력 조건

| 항목 | 정책 |
| --- | --- |
| 길이 | 8자 이상 20자 이하 |
| 조합 | 영문, 숫자, 특수문자 중 2가지 이상 조합 |
| 공백 | 입력 불가 |
| 기존 비밀번호 | 기존 비밀번호와 동일하게 설정 불가 |

### 4.2 확인 비밀번호 조건

| 조건 | 메시지 |
| --- | --- |
| 새 비밀번호와 불일치 | 비밀번호가 일치하지 않아요. |
| 미입력 | 비밀번호를 한 번 더 입력해 주세요. |

## 5. 인증 링크 정책

| 항목 | 정책 |
| --- | --- |
| 유효 시간 | 발송 시점부터 30분 |
| 재사용 | 1회 사용 후 즉시 만료 |
| 재발송 제한 | 1분 내 재발송 불가 |
| 일일 요청 제한 | 동일 이메일 기준 1일 5회 |
| 링크 만료 처리 | 비밀번호 찾기 화면으로 재진입 유도 |

## 6. 상태 및 메시지 정의

### 6.1 토스트 메시지

| 상황 | 메시지 |
| --- | --- |
| 메일 발송 성공 | 인증 메일을 발송했어요. |
| 재발송 제한 | 잠시 후 다시 시도해 주세요. |
| 서버 오류 | 일시적인 오류가 발생했어요. 다시 시도해 주세요. |
| 비밀번호 변경 성공 | 비밀번호가 변경되었어요. |

### 6.2 화면 메시지

| 화면 | 메시지 |
| --- | --- |
| 이메일 인증 안내 | 입력한 이메일로 비밀번호 재설정 링크를 보냈어요. |
| 링크 만료 | 인증 시간이 만료되었어요. 다시 요청해 주세요. |
| 완료 | 새 비밀번호로 로그인해 주세요. |

## 7. API 연동 정의

### 7.1 비밀번호 재설정 메일 요청

| 항목 | 내용 |
| --- | --- |
| Method | POST |
| Endpoint | `/api/v1/auth/password-reset/request` |
| Request | `email` |
| Response | `requestId`, `sentAt` |
| 비고 | 가입되지 않은 이메일도 성공 응답과 동일하게 처리 |

### 7.2 토큰 검증

| 항목 | 내용 |
| --- | --- |
| Method | GET |
| Endpoint | `/api/v1/auth/password-reset/verify` |
| Request | `token` |
| Response | `valid`, `expiredAt` |
| 비고 | 만료 또는 사용 완료 토큰은 `valid=false` |

### 7.3 새 비밀번호 저장

| 항목 | 내용 |
| --- | --- |
| Method | POST |
| Endpoint | `/api/v1/auth/password-reset/confirm` |
| Request | `token`, `newPassword` |
| Response | `success` |
| 비고 | 성공 시 기존 로그인 세션 전체 만료 |

## 8. 권한 및 보안 요구사항

- 비밀번호 재설정 링크에는 사용자 식별 정보를 직접 포함하지 않는다.
- 토큰은 추측 불가능한 난수로 생성한다.
- 토큰은 서버에서 해시 처리하여 저장한다.
- 비밀번호 변경 완료 시 기존 로그인 세션은 모두 만료한다.
- 비밀번호 입력값은 로그에 남기지 않는다.

## 10. 완료 기준

- 사용자는 가입 이메일을 통해 비밀번호 재설정 링크를 받을 수 있다.
- 사용자는 유효한 링크로 새 비밀번호를 설정할 수 있다.
- 만료, 재사용, 잘못된 토큰에 대한 예외 처리가 동작한다.
- 비밀번호 정책에 맞지 않는 입력은 저장되지 않는다.
- 비밀번호 변경 후 기존 세션은 만료된다.
- QA 체크리스트의 필수 항목이 모두 통과한다.
